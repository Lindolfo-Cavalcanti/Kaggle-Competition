Introduction

The Spaceship Titanic competition on Kaggle presents an intriguing challenge inspired by the classic Titanic disaster but set in a futuristic context. Participants are tasked with predicting whether passengers were transported to an alternate dimension during an interstellar voyage. The competition’s dataset includes a variety of features about the passengers, such as their demographics, cabin assignments, and spending habits on board.

This report aims to provide a comprehensive analysis of the Spaceship Titanic dataset, encompassing data exploration, visualization, and predictive modeling. By leveraging these techniques, I seek to uncover patterns and insights that can enhance our understanding of the factors influencing passenger transportation.

Objectives

  1. Data Exploration:
    • Conduct a thorough examination of the dataset to understand its structure and identify key features.
    • Handle missing values and perform necessary data cleaning.
    • Explore the distributions and relationships of variables to gain initial insights.
  2. Data Visualization:
    • Utilize visual techniques to represent the data graphically, revealing underlying patterns and trends.
    • Create plots and charts that illustrate the relationships between different features and the target variable (transported status).
  3. Predictive Modeling:
    • Develop and train machine learning models to predict the transportation status of passengers.
    • Evaluate the performance of various models using appropriate metrics.
    • Optimize the models to achieve the best possible predictive accuracy.

Packages Overview

tidyverse:

A collection of R packages designed for data science that Facilitates workflow for data manipulation, exploration, and visualization.

randomForest:

An implementation of the Random Forest algorithm, useful for both classification and regression tasks. Helps build an ensemble model that improves prediction accuracy by averaging multiple decision trees.

dlookr:

A package for data diagnosis, transformation, and visualization. Assists in identifying missing values, outliers, and understanding the distribution of data.

flextable:

Provides a straightforward way to create and customize tables in R, suitable for reporting and documentation. Enhances the presentation of data summaries and model results.

esquisse:

An interactive tool that simplifies the process of creating ggplot2 visualizations without writing extensive code. Enables users to drag-and-drop variables to create plots, making data visualization more accessible.

ggThemeAssist:

A graphical user interface add-in for customizing ggplot2 themes. Allows for easy adjustment of plot aesthetics to improve the visual appeal of data visualizations.

As observed from the initial analysis, the results for various spending categories on the spaceship are quite different from those related to age. This discrepancy is likely due to the fact that these categories represent different aspects of money spent on the spaceship, which can vary significantly. To address this issue, we should combining these similar spending categories into a new aggregated category. This approach will help us simplify the analysis and draw more meaningful insights by reducing the variability introduced by multiple related features.

But, before doing that let’s deal with the missing values.

median_na_solution = function(x) {
    x[is.na({{x}})] = median({{x}}, na.rm = TRUE)
    return(x)
}

    full_space$RoomService[is.na(full_space$RoomService)] = median_na_solution(full_space$RoomService)
    full_space$FoodCourt[is.na(full_space$FoodCourt)] = median_na_solution(full_space$FoodCourt)
    full_space$ShoppingMall[is.na(full_space$ShoppingMall)] = median_na_solution(full_space$ShoppingMall)
    full_space$Spa[is.na(full_space$Spa)] = median_na_solution(full_space$Spa)
    full_space$VRDeck[is.na(full_space$VRDeck)] = median_na_solution(full_space$VRDeck)
